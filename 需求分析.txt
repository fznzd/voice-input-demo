                    语音输入法产品 — 需求分析文档
================================================================================
版本：v1.0
日期：2026-05-23
项目：七牛云项目-语音输入法

================================================================================
一、项目背景与目标
================================================================================

1.1 背景
  用户在 PC 端进行文字输入时，键盘打字效率低、长段落输入疲劳。语音输入可
  显著提升输入效率，尤其适用于办公写作、即时通讯、表单填写等场景。

  当前项目已验证「语音识别 → 文字」的技术链路，但尚未具备输入法核心能力
  （实时采集、流式识别、全局上屏）。

1.2 产品定位
  面向 Windows 桌面用户的语音输入法工具：通过全局快捷键唤起，按住说话，
  实时转写并将文字插入当前焦点应用。

1.3 核心目标
  - 提高文本输入效率，减少键盘依赖
  - 在准确度、易用性、响应速度、成本之间取得平衡
  - 支持中文普通话为主，输出简体中文
  - 可扩展长语音转写等增值能力（对接七牛 LASR）

1.4 非目标（首期不做）
  - 不做系统级 IME 内核（Android/iOS 输入法 SDK）
  - 不做多用户协作、云端文档编辑
  - 不做实时同声传译

二、目标用户与典型场景
================================================================================

2.1 目标用户
  ┌──────────────┬────────────────────────────────────────────────┐
  │ 用户类型     │ 特征与诉求                                     │
  ├──────────────┼────────────────────────────────────────────────┤
  │ 办公白领     │ 写邮件、周报、文档；需要标点、段落、少改错     │
  │ 内容创作者   │ 长段口述转文字；关注准确率与连续听写           │
  │ 开发者/运维  │ 快速输入命令说明、日志备注；术语多             │
  │ 中老年用户   │ 不熟悉键盘；需要简单交互、清晰反馈             │
  │ 轻度障碍用户 │ 打字困难；依赖语音作为主要输入方式             │
  └──────────────┴────────────────────────────────────────────────┘

2.2 典型应用场景
  场景 A：即时通讯
    - 用户在微信/钉钉中回复消息
    - 按住快捷键说话，松手后文字自动上屏发送
    - 要求：低延迟、短句准确、支持口语化表达

  场景 B：文档写作
    - 用户在 Word/Notion/浏览器编辑器中撰写长文
    - 连续听写多句，自动加标点与换行
    - 要求：准确率优先，支持语音命令（换行、删除）

  场景 C：表单与搜索
    - 用户在网页表单、搜索框中输入
    - 短句快速输入，即说即走
    - 要求：首字延迟低，插入位置正确

  场景 D：专业领域输入（进阶）
    - 医疗、法律、客服等含大量专有名词
    - 要求：热词/个人词典，领域术语识别准确

  场景 E：长录音转写（增值）
    - 用户上传会议录音或较长备忘录
    - 异步转写，不要求实时上屏
    - 要求：对接七牛 LASR，支持长音频、标点


================================================================================
三、用户痛点与产品价值
================================================================================

3.1 痛点
  - 键盘输入慢，长文写作耗时长
  - 现有语音输入多绑定在单一 App 内，无法全局使用
  - 通用 ASR 对专有名词、人名识别差
  - 部分工具延迟高，「说完再等」体验差
  - 云端 ASR 按量计费，无 VAD 时成本易失控

3.2 产品价值
  - 任意应用内一键语音输入，无需切换窗口
  - 流式识别，边说边看到文字，降低等待焦虑
  - 热词与个人词典提升专业场景准确率
  - 端侧 VAD + 按住说话，控制成本
  - 长转写能力满足会议/备忘录等延伸需求

四、功能需求
================================================================================

4.1 核心功能（MVP 必须）

  F1 全局唤起
    - 支持全局快捷键（建议：Win+Shift+V 按住说话）
    - 系统托盘图标，右键菜单（设置、退出、用量）
    - 启动后后台常驻，资源占用可控

  F2 语音采集
    - 从默认麦克风采集 16kHz / 16bit / 单声道 PCM
    - 按住说话、松手结束；可选连续听写模式
    - 端侧 VAD：静音段不上传，降低无效计费

  F3 流式语音识别
    - WebSocket 连接云端流式 ASR（如 fun-asr-realtime）
    - 接收 partial（中间结果）与 final（最终结果）
    - 支持中文普通话，默认输出简体中文
    - 网络异常时给出明确提示，支持重试

  F4 文本上屏
    - 将 final 文本插入当前焦点应用的光标位置
    - 实现方式：模拟键盘输入（优先）或剪贴板粘贴（备选）
    - 支持 Word、浏览器、微信、记事本等常见应用

  F5 基础 UI 反馈
    - 悬浮窗或迷你条：显示录音状态、音量、实时识别预览
    - 识别中 / 识别完成 / 错误 等状态可视化

  F6 基础设置
    - 快捷键自定义
    - 麦克风设备选择
    - ASR 服务配置（API Key 等，经后端代理，不暴露在前端）

4.2 体验增强功能（Phase 2）

  F7 标点与后处理
    - 自动添加句号、逗号、问号等
    - 数字、日期规范化（如「二零二六年」→「2026年」）
    - 口语顺滑（去除「嗯、啊」等，可配置）

  F8 语音命令
    - 「换行」「删除」「逗号」「句号」等口令
    - 无需手触键盘即可完成简单编辑

  F9 热词与个人词典
    - 用户维护词条列表（人名、项目名、术语）
    - 同步至 ASR 热词接口，提升识别率

  F10 历史与纠错
    - 本地保存最近 N 条识别记录（可选，注意隐私）
    - 一键重新识别上一段

4.3 增值功能（Phase 3）

  F11 长语音转写
    - 对接七牛 LASR，支持上传/录制长音频
    - 异步任务 + 进度查询 + 结果导出

  F12 多语言与方言（可选）
    - 英语、粤语等，按 ASR 供应商能力扩展

  F13 用量统计与计费
    - 展示本月已用时长/次数
    - 支持套餐或按量限制

五、非功能需求
================================================================================

5.1 准确度
  - 中文普通话（安静环境）：字准确率 ≥ 95%
  - 专有名词（配置热词后）：显著提升，目标 ≥ 90%
  - 默认输出简体中文

5.2 响应速度
  - 首字延迟（按下到首个 partial）：< 500ms（目标 < 300ms）
  - 松手到 final 上屏：< 800ms
  - 音频 chunk 上传间隔：200~300ms

5.3 易用性
  - 首次使用：3 步内完成（安装 → 授权麦克风 → 试说一次）
  - 无需切换目标应用即可输入
  - 错误信息可理解（网络、权限、设备占用等）

5.4 可靠性
  - 连续使用 2 小时无崩溃
  - 网络闪断可自动重连或 graceful 降级
  - 不因单次识别失败导致进程退出

5.5 成本
  - 端侧 VAD 减少 30%~50% 无效上传
  - 按住说话模式优先于 7×24 连续监听
  - 后端代理 ASR Key，支持用量限流

5.6 安全与隐私
  - 音频默认不落盘；若落盘需用户知情并可关闭
  - API Key 仅存服务端，客户端经鉴权访问
  - 符合最小必要原则收集用户数据

5.7 兼容性
  - 首期：Windows 10/11
  - 麦克风：系统默认及可选列表
  - 目标应用：主流办公、浏览器、IM 客户端

七、技术架构要点（需求层面）
================================================================================

  客户端（Windows）
    - 采集层：麦克风 + VAD
    - 通信层：WebSocket 流式上传音频
    - 交互层：全局快捷键 + 悬浮 UI
    - 插入层：键盘模拟 / 剪贴板

  服务端（轻量）
    - 用户鉴权、会话管理
    - ASR 代理（隐藏 Key、限流、日志）
    - 热词存储与同步

  ASR 策略
    - 实时输入：流式 ASR（阿里 fun-asr-realtime / 腾讯 WebSocket）
    - 长转写：七牛 LASR（异步，非输入法主链路）
    - 离线兜底（可选）：本地 Whisper，延迟高，仅弱网备用

  与当前 Demo 的差异
    ┌─────────────┬──────────────────┬──────────────────────────┐
    │ 能力        │ 当前 Demo        │ 输入法目标               │
    ├─────────────┼──────────────────┼──────────────────────────┤
    │ 输入        │ 音频文件         │ 麦克风实时流             │
    │ ASR         │ 离线批处理       │ 云端流式                 │
    │ 输出        │ 终端打印         │ 全局上屏                 │
    │ 交互        │ 命令行           │ 快捷键 + 悬浮 UI         │
    │ 延迟        │ 秒级             │ 亚秒级                   │
    └─────────────┴──────────────────┴──────────────────────────┘

八、用户故事（User Stories）
================================================================================

  US-01  作为办公用户，我希望按住快捷键说话，以便在写邮件时不用停下来打字。
  US-02  作为 IM 用户，我希望松手后文字立刻出现在聊天框，以便快速回复消息。
  US-03  作为写作者，我希望看到实时识别预览，以便确认模型「听对了」。
  US-04  作为开发者，我希望添加项目专有名词到热词，以便 ASR 正确识别库名/接口名。
  US-05  作为用户，我希望说「换行」时自动换行，以便少碰键盘。
  US-06  作为用户，当网络断开时，我希望看到明确提示，以便知道如何恢复。
  US-07  作为管理员，我希望 API Key 不在客户端明文存储，以便降低泄露风险。
  US-08  作为用户，我希望上传一段会议录音转文字，以便做会议纪要（增值）。

九、分阶段交付计划
================================================================================

  Phase 1 — MVP（4~6 周）
    - 麦克风采集 + VAD
    - 流式 ASR 对接
    - 全局快捷键 + 文本上屏
    - 基础悬浮 UI + 托盘
    - 验收：3 个典型 App 内稳定输入

  Phase 2 — 体验优化（2~3 周）
    - 标点/数字后处理
    - 语音命令
    - 热词与个人词典
    - 设置面板完善

  Phase 3 — 平台与商业化（按需）
    - 七牛 LASR 长转写
    - 用量统计与套餐
    - macOS 版本评估

十、风险与对策
================================================================================

  风险                          影响        对策
  ─────────────────────────────────────────────────────────────
  流式 ASR 供应商选型未定       进度        抽象 ASRProvider，便于切换
  键盘模拟被安全软件拦截        上屏失败    提供剪贴板模式；签名/白名单指引
  嘈杂环境准确率低              体验差      提示佩戴耳机；端侧降噪
  云端 ASR 成本超预期           商业        VAD + 按住说话 + 用量上限
  Whisper 离线延迟高            不适合 IME  仅作弱网兜底，不作主链路
  简繁混输出                    大陆用户    initial_prompt 引导简体

十一、附录：标准测试用例（建议）
================================================================================

  1. 短句测试（10 字以内）：「今天天气很好」
  2. 长句测试（30~50 字）：办公邮件常用句式
  3. 标点测试：含问号、逗号、顿号的口语句
  4. 数字测试：「我的手机号是一三八开头」
  5. 热词测试：含 3~5 个项目专有名词的句子
  6. 上屏测试：Word / Chrome 输入框 / 微信 各 5 次
  7. 弱网测试：限速 100KB/s 下是否可完成识别
  8. 连续测试：连续输入 20 句无崩溃
