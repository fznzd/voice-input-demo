    "input_mode": "push_to_talk",       // push_to_talk | continuous

    "backend_token": "<encrypted or empty>",
    "language": "zh",
    "simplified_prompt": "以下是普通话的句子。",
    "vad_aggressiveness": 2,            // 0~3
    "chunk_ms": 200,
    "hotwords": ["七牛云", "语音输入法"],
    "enable_postprocess": true,
    "enable_voice_commands": true,
    "overlay_enabled": true
  }

2.2 会话 SessionContext

  @dataclass
  class SessionContext:
      session_id: str              # UUID，全链路日志关联
      started_at: float            # time.monotonic()
      state: SessionState          # 枚举见 2.3
      partial_text: str            # 当前预览文本
      final_segments: list[str]    # 已确认分段
      audio_frames_sent: int       # 已发送帧数（统计/计费）
      error: Optional[str]

2.3 会话状态 SessionState

  IDLE = "idle"
  RECORDING = "recording"
  RECOGNIZING = "recognizing"
  FLUSHING = "flushing"            # 松手后等待 ASR 收尾
  INSERTING = "inserting"
  ERROR = "error"

2.4 ASR 事件 ASREvent

  {
    "type": "partial" | "final" | "error" | "ready",
    "text": "识别文本",
    "start_ms": 0,
    "end_ms": 2500,
    "confidence": 0.95,            // 可选
    "session_id": "uuid",
    "seq": 12                      // 序号，防乱序
  }

2.5 热词 HotwordEntry

  {
    "word": "七牛云",
    "weight": 10                     // 1~100，供应商格式适配层转换
  }

3.1 InputOrchestrator（会话编排器）
────────────────────────────────────────────────────────────────

  职责：协调一次语音输入的完整生命周期，维护 SessionContext 状态机。

  类接口：
    class InputOrchestrator:
        def __init__(self, config, audio, vad, asr, postprocess,
                     commands, inserter, ui, logger)
        def on_hotkey_down(self) -> None      # 快捷键按下
        def on_hotkey_up(self) -> None        # 快捷键松开
        def start(self) -> None               # 初始化后台线程
        def stop(self) -> None                # 退出清理

  内部流程 — on_hotkey_down：
    1. 若 state != IDLE，忽略或先 cancel 上一会话
    2. 创建 SessionContext，state = RECORDING
    3. ui.show_recording()
    4. audio.start(callback=on_audio_frame)
    5. asr.connect(session_id)
    6. state = RECOGNIZING

  内部流程 — on_audio_frame(pcm_bytes)：
    1. 若 vad.is_speech(pcm_bytes) 为 False，跳过（不上传）
    2. asr.send_audio(pcm_bytes)
    3. ctx.audio_frames_sent += 1

  内部流程 — on_asr_partial(text)：
    1. ctx.partial_text = postprocess.preview(text)
    2. ui.update_preview(ctx.partial_text)

  内部流程 — on_asr_final(text)：
    1. processed = postprocess.finalize(text)
    2. action = commands.parse(processed)   # 普通文本 | 命令
    3. 若命令：inserter.execute(action)
       否则：inserter.insert_text(action.text); ctx.final_segments.append(...)
    4. ctx.partial_text = ""
    5. ui.update_preview("")

  内部流程 — on_hotkey_up：
    1. audio.stop()
    2. state = FLUSHING
    3. asr.finish()                         # 发送结束信号
    4. 等待 final 或超时（默认 3s）
    5. state = IDLE and INSERTING → IDLE
    6. ui.hide_recording()

  超时与错误：
    - ASR 连接失败：重试 3 次，间隔 1s/2s/4s
    - 3s 无 final：使用最后 partial 兜底上屏（可配置）
    - 麦克风占用：ui.show_error("麦克风被占用")


3.2 AudioCapture（音频采集）
────────────────────────────────────────────────────────────────

  参数：
    sample_rate = 16000
    channels = 1
    dtype = int16
    blocksize = int(sample_rate * chunk_ms / 1000)   # 200ms → 3200 samples

  类接口：
    class AudioCapture:
        def list_devices(self) -> list[DeviceInfo]
        def start(self, on_frame: Callable[[bytes], None]) -> None
        def stop(self) -> None

  实现要点：
    - 使用 sounddevice InputStream，callback 模式
    - callback 内只做 bytes 转换，不做阻塞 IO
    - 帧格式：raw PCM little-endian int16


3.3 VoiceActivityDetector（VAD）
────────────────────────────────────────────────────────────────

  类接口：
    class VoiceActivityDetector:
        def __init__(self, sample_rate=16000, aggressiveness=2)
        def is_speech(self, pcm_frame: bytes) -> bool
        def reset(self) -> None

  实现（webrtcvad）：
    - 帧长必须 10/20/30 ms；200ms 块切分为 20ms × 10 帧
    - 任一致帧为 speech 则该 chunk 上传（减少漏字）
    - 或：≥ 30% 帧为 speech 才上传（更省流量，可配置）

  参数建议：
    chunk_ms=200, frame_ms=20, speech_ratio_threshold=0.1


3.4 ASRClient 抽象层与阿里云实现
────────────────────────────────────────────────────────────────

  3.4.1 抽象基类 asr/base.py

    class ASRProvider(ABC):
        @abstractmethod
        async def connect(self, session_id: str, options: ASROptions) -> None
        @abstractmethod
        async def send_audio(self, pcm: bytes) -> None
        @abstractmethod
        async def finish(self) -> None
        @abstractmethod
        async def close(self) -> None

        # 回调注册
        on_partial: Callable[[str], None]
        on_final: Callable[[str], None]
        on_error: Callable[[Exception], None]

  3.4.2 ASROptions

    @dataclass
    class ASROptions:
        language: str = "zh"
        sample_rate: int = 16000
        hotwords: list[str] = field(default_factory=list)
        enable_punctuation: bool = True
        format: str = "pcm"

  3.4.3 经后端代理的 WebSocket 协议（客户端 ↔ 后端）

    连接：wss://{host}/ws/asr?token={jwt}

    客户端 → 服务端（Text JSON）：
      开始：
        {"action":"start","session_id":"uuid","options":{...}}
      结束：
        {"action":"finish","session_id":"uuid"}
      心跳：
        {"action":"ping"}

    客户端 → 服务端（Binary）：
      原始 PCM 帧，无封装头

    服务端 → 客户端（Text JSON）：
      {"type":"partial","text":"你好","seq":1}
      {"type":"final","text":"你好，世界。","seq":2}
      {"type":"error","code":"ASR_TIMEOUT","message":"..."}
      {"type":"ready"}

  3.4.4 后端 ws_proxy 职责
    - 校验 JWT
    - 建立到阿里云 fun-asr-realtime 的上游 WebSocket
    - 双向转发音频与文本
    - 记录 session 时长（用量）
    - 限流：单用户并发 session ≤ 1，每分钟 chunk 上限


3.5 PostProcessor（后处理管道）
────────────────────────────────────────────────────────────────

  管道顺序（可配置）：
    raw_text
      → trim_whitespace
      → apply_simplified_prompt_effect（已在 ASR 层）
      → normalize_numbers（Phase 2）
      → normalize_punctuation（Phase 2）
      → remove_filler_words（嗯、啊，可选）

  类接口：
    class PostProcessor:
        def preview(self, text: str) -> str    # partial 预览用，轻量
        def finalize(self, text: str) -> str     # final 上屏用，完整管道

  MVP：preview 与 finalize 均可直接返回 strip 后文本


3.6 CommandParser（语音命令）
────────────────────────────────────────────────────────────────

  命令表（Phase 2，MVP 可 stub）：
    口令            动作                上屏行为
    ─────────────────────────────────────────────────
    "换行"          INSERT_NEWLINE      发送 Enter
    "删除"          DELETE_LAST         发送 Backspace×N 或 Ctrl+Z
    "逗号"          INSERT_CHAR         插入 ，
    "句号"          INSERT_CHAR         插入 。
    "问号"          INSERT_CHAR         插入 ？

  类接口：
    @dataclass
    class CommandAction:
        kind: Literal["text","newline","delete","char"]
        text: str = ""

    class CommandParser:
        def parse(self, text: str) -> CommandAction

  匹配规则：final 文本完全等于命令词，或「命令词。」形式


3.7 TextInserter（文本上屏）
────────────────────────────────────────────────────────────────

  类接口：
    class TextInserter:
        def __init__(self, mode: Literal["keyboard","clipboard"])
        def insert_text(self, text: str) -> None
        def insert_newline(self) -> None
        def delete_chars(self, n: int) -> None
        def execute(self, action: CommandAction) -> None

  keyboard 模式实现：
    - 使用 pynput.keyboard.Controller
    - 中文：逐字 type 或使用 pyperclip 粘贴（中文兼容性更好）
    - 推荐：中文文本用 clipboard 粘贴，英文/数字用 type
    - 粘贴流程：备份剪贴板 → 写入文本 → Ctrl+V → 恢复剪贴板

  焦点保持：
    - 按下快捷键前记录 hwnd（Win32 GetForegroundWindow）
    - 上屏前 SetForegroundWindow 还原焦点（若被悬浮窗抢走）

  防重复：同 session 同一 final 文本不重复插入（hash 去重）

5.1 WebSocket 端点 /ws/asr

  依赖：verify_jwt(token)

  伪代码：
    @app.websocket("/ws/asr")
    async def asr_ws(websocket, token: str):
        user = verify_jwt(token)
        await websocket.accept()
        upstream = await connect_aliyun_asr(user.hotwords)
        try:
            async for msg in websocket.iter():
                if isinstance(msg, bytes):
                    await upstream.send_audio(msg)
                else:
                    data = json.loads(msg)
                    if data["action"] == "start":
                        await upstream.start(data["options"])
                    elif data["action"] == "finish":
                        await upstream.finish()
            async for result in upstream.results():
                await websocket.send_json(result)
                if result["type"] == "final":
                    usage.record(user.id, duration)
        finally:
            await upstream.close()

5.2 鉴权

六、线程与并发模型
================================================================================

  线程/任务              职责                    阻塞要求
  ─────────────────────────────────────────────────────────────
  主线程 (UI)            Tk/Qt 事件循环           不阻塞
  热键监听线程           pynput 回调              仅发事件到队列

  ASR asyncio loop       WebSocket IO             独立线程跑 loop

  模块间通信：threading.Queue / asyncio.Queue
  事件类型：HotkeyDown, HotkeyUp, AudioFrame, ASRPartial, ASRFinal, Error

七、错误码与处理
================================================================================

  错误码              场景                    用户提示
  ─────────────────────────────────────────────────────────────
  MIC_NOT_FOUND       无麦克风设备            请连接麦克风
  MIC_BUSY            设备被占用              麦克风被其他程序占用
  ASR_CONNECT_FAIL    WS 连接失败             无法连接语音服务
  ASR_TIMEOUT         收尾超时                识别超时，请重试
  INSERT_FAIL         上屏失败                无法写入当前应用，尝试剪贴板模式
  AUTH_FAIL           Token 无效              请重新登录
  QUOTA_EXCEEDED      用量超限                本月用量已达上限

  日志格式：
    [2026-05-23 10:00:01] [session=abc-123] [INFO] ASR connected
    [2026-05-23 10:00:03] [session=abc-123] [ERROR] ASR_TIMEOUT
